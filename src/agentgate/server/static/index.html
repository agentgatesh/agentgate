What is AgentGate?

Think of AgentGate as a phone book and switchboard for AI agents.

The problem

Today, AI agents live in silos. Each one is built differently, speaks a different language, and has no easy way to find or talk to other agents. If your travel-booking agent needs to check a weather agent, there's no standard way to make that happen.

The solution

AgentGate gives every agent a public profile (called an Agent Card), a standard way to communicate, and a place where other agents (and people) can find them. Deploy once, connect everywhere.

How It Works

Three steps. No PhD required.

1

Describe your agent

Create a simple agentgate.yaml file that says what your agent does, what skills it has, and where it lives. That's it — a few lines of text.

2

Deploy with one command

Run agentgate deploy ./my-agent from your terminal. Your agent is now registered on the gateway and gets its own Agent Card — a standardized profile that any other system can read.

3

Connect and discover

Other agents and applications can find yours through the gateway. They read the Agent Card, understand what your agent can do, and start collaborating — automatically.

Features

Everything you need to get your agents connected.

One-command deploy

Write a YAML file, run one command. Your agent is live and discoverable.

A2A Agent Cards

Every agent gets a standardized profile following the Agent-to-Agent protocol.

Agent-to-Agent routing

Route tasks between agents via the gateway. Chain agents for complex workflows.

Per-agent auth

Protect individual agents with their own API keys. Gateway-level and agent-level security.

Invocation logs

Every task is logged with caller IP, latency, and status. Full audit trail.

Usage tracking

Monitor invocations per agent. Prepare for billing and monetization.

Health monitoring

Background health checks every 60s. Know when agents go down before your users do.

Webhooks

Get notified when tasks complete. Fire-and-forget callbacks to any URL.

Live dashboard

Real-time metrics dashboard with latency charts and error tracking.

Discovery endpoint

Standard /.well-known/agent.json endpoint for automatic agent discovery.

Python SDK

pip install agentgatesh — full client for registry, tasks, logs, and health.

Monetization

Set a price per task, manage org wallets, and earn from your agents with built-in billing. See pricing.

Open source

AGPL licensed. Run it yourself, contribute, or build on top of it.
